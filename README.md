# MoE for Multilingual Semantic Search

Реализация модели семантического поиска на архитектуре Mixture of Experts (MoE) на базе multilingual-E5-large для многоязычных задач retrieval.

## Структура репозитория

- `modeling_xlm_roberta_moe.py` – реализация MoE-архитектуры на основе XLM-RoBERTa
- `train_moe.py` – скрипт обучения модели на мультиязычных данных
- `evaluate.py` – скрипт оценки качества моделей

## Описание

### MoE архитектура

Файл `modeling_xlm_roberta_moe.py` содержит реализацию MoE-архитектуры, где каждый Feed-Forward слой заменён на набор экспертов. Каждый эксперт представляет собой полносвязную сеть с подъёмной и проекционной матрицами. Модель поддерживает жёсткий выбор эксперта через тензор `expert_paths` размерности `[batch, seq_len, num_layers]`, что позволяет привязать определённого эксперта к конкретному языку. Инициализация происходит из плотной модели multilingual-E5-large: веса эмбеддингов, внимания и слоёв нормализации загружаются из предобученной модели, все эксперты внутри одного MoE-слоя копируют веса исходного плотного FFN, а роутер обнуляется, чтобы изначально не отдавать предпочтение ни одному из экспертов. Модель совместима с библиотекой transformers и поддерживает сохранение и загрузку чекпоинтов.

### Обучение

Файл `train_moe.py` обеспечивает цикл обучения MoE-модели. На вход подаётся конфигурационный JSON-файл с указанием путей к данным с разбивкой по языкам и параметрами обучения: размер батча, скорость обучения, количество шагов прогрева, градиентная аккумуляция, периодичность сохранения чекпоинтов и валидации. Поддерживаются два формата данных: JSON-файлы с полями `query`, `pos` и опционально `neg`, а также папки HuggingFace datasets, содержащие `corpus` с идентификаторами и текстами документов и `dataset` с запросами, позитивными и негативными документами. При формировании батча специальный коллатор группирует примеры по языкам и для каждого токена создаёт тензор `expert_paths`, который на всех слоях модели направляет токены данного языка к эксперту, соответствующему этому языку. Функция потерь InfoNCE объединяет косинусную близость между запросом и положительным документом, близость с другими документами внутри батча и близость с явными негативными примерами. Обучение ведётся с градиентной аккумуляцией, warmup и линейным расписанием скорости обучения. Регулярно выполняется валидация на отложенных данных с вычислением метрик nDCG@10, MRR@10, P@10 и Recall@10, лучший чекпоинт сохраняется отдельно. При указании флага `--log_to_clearml` все метрики и loss логируются в ClearML.

### Оценка

Файл `evaluate.py` предназначен для оценки качества моделей и поддерживает три режима. Режим `base` использует исходную плотную модель multilingual-E5-large без MoE. Режим `moe_init` использует только что инициализированную MoE-модель без обучения. Режим `moe_saved` загружает обученную MoE-модель с диска по пути, переданному в аргументе `--checkpoint`. Для каждого тестового датасета программа загружает corpus и запросы, кодирует их с использованием префиксов `query` или `passage`, а для MoE-вариантов дополнительно формирует `expert_paths` на основе языка текущего датасета, чтобы все токены обрабатывались экспертом, соответствующим этому языку. После получения эмбеддингов вычисляются косинусные сходства между всеми запросами и документами, и по полученным ранг-листам с помощью библиотеки ir_measures рассчитываются метрики nDCG@10, MRR@10, P@10 и Recall@10. Результаты сохраняются в JSON-файл.

## Использование

### Подготовка конфигурации

Создайте JSON-конфиг для обучения:

{
  "data_config": {
    "en": ["/path/to/en/data"],
    "ru": ["/path/to/ru/data.json"]
  },
  "output_dir": "./output",
  "model_config_path": "./moe_config.json",
  "batch_size": 8,
  "grad_acc_steps": 4,
  "learning_rate": 2e-5,
  "max_epochs": 3,
  "save_steps": 5000,
  "eval_steps": 1000,
  "max_length": 512,
  "seed": 42
}

Создайте JSON-конфиг модели:

{
  "num_hidden_layers": 24,
  "hidden_size": 1024,
  "intermediate_size": 4096,
  "num_attention_heads": 16,
  "moe_num_experts": 8
}

### Запуск обучения

python train_moe.py --config config.json --model_config moe_config.json

Дополнительные опции:

python train_moe.py --config config.json --model_config moe_config.json --batch_size 4 --learning_rate 1e-5 --max_epochs 5 --output_dir ./my_experiment --log_to_clearml --project_name "Semantic Search" --experiment_name "moe_experiment_1"

### Запуск оценки

Оценка базовой модели:

python evaluate.py --config config.yaml --model_type base

Оценка инициализированной MoE:

python evaluate.py --config config.yaml --model_type moe_init

Оценка обученной MoE из чекпоинта:

python evaluate.py --config config.yaml --model_type moe_saved --checkpoint ./output/best

## Формат данных

### JSON формат

Файл должен содержать список объектов:

[
  {
    "query": "текст запроса",
    "pos": "релевантный документ",
    "neg": "нерелевантный документ"
  },
  {
    "query": "другой запрос",
    "pos": "релевантный документ",
    "neg": "нерелевантный документ"
  }
]

Поле neg является опциональным.

### HuggingFace datasets

Структура папок:

/path/to/data/
├── corpus/
│   ├── dataset.arrow
│   └── dataset_info.json
└── dataset/
    ├── dataset.arrow
    └── dataset_info.json

Корпус должен содержать поля id и text. Датасет должен содержать поля query, positives (словарь с ключом doc_id и списком идентификаторов), опционально negatives (словарь с ключом doc_id и списком идентификаторов).

## Метрики

Оценка производится по следующим метрикам: nDCG@10, MRR@10, P@10 (Precision at 10), R@10 (Recall at 10).

## Требования

Python 3.8 или выше, PyTorch, transformers, datasets, ir_measures, tqdm, clearml (опционально).
